## 抽象工厂模式（Abstract Factory Pattern）

定义：为创建一组相关或相互依赖的对象提供一个接口，而且无需指定他们的具体类

#### 抽象工厂模式通用代码

```
//抽象产品类
public abstract class AbstractProductA {

    public void shareMethod() {
    }
    public abstract void doSomething();
}

public abstract class AbstractProductB {
    ......
}
//产品A1的实现类
public class ProductA1 extends AbstractProductA {
    public void doSomething() {
        System.out.println("productA method");
    }
}
//产品A2的实现类
public class ProductA2 extends AbstractProductA {
    public void doSomething() {
        System.out.println("priductA2 method");
    }
}
//抽象工厂类
public abstract class AbstractCreator {
    public abstract AbstractProductA createPriductA();
    public abstract AbstractProductB createPriductB();
}
//产品等级1的实现类
public class Creator1 extends AbstractCreator{
    //只生产产品等级A1的产品
    public AbstractProductA createPriductA() {
        return new ProductA1();
    }
    //只生产产品等级A2的产品
    public AbstractProductB createPriductB() {
        return new ProductB1();
    }
}
//产品等级2的实现类
public class Creator2 extends AbstractCreator {
    public AbstractProductA abstractProductA() {
        return new ProductA1();
    }
    public AbstractProductB abstractProductB() {
        return new ProductB1();
    }
}

//场景类
public class Client {
    public static void main(String[] args) {
        //定义出两个工厂
        AbstractCreator creator1 = new Creator1();
        AbstractCreator creator2 = new Creator2();
        //产生A1对象
        AbstractProductA a1 = creator1.createPriductA();
        //产生A2对象
        AbstractProductA a2 = creator2.createPriductA();
        //产生B1对象
        AbstractProductB b1 = creator2.createPriductB();
        //产生B2对象
        AbstractProductB b2 = creator2.createPriductB();
        ......
    }
}
在场景类中没有任何一个方法与实现类有关系，对于一个产品来说 只需要知道它的工厂方法就可以直接产生一个产品对象，无需关心它的实现类
```

#### 抽象工厂模式的优点

1.封装性，产品的实现类不需要高层模块来关心

2.约束状态为非公开


#### 抽象工厂模式的缺点

1.产品族扩展非常困难
